Como ya he dicho están escritos en algo parecido a C. Siempre se empieza con una declaración de la versión. 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\#version\ 330\ }\textcolor{comment}{//\ Equivale\ a\ OpenGL\ 3.3}}

\end{DoxyCode}


Tiene unas variables para comunicarnos con el shader:\+
\begin{DoxyItemize}
\item Inputs:\+ sirven para recibir información de ese vértice en concreto. Por ejemplo:\+ color, posición textura... 
\begin{DoxyCode}{0}
\DoxyCodeLine{in\ vec3\ aPos;}

\end{DoxyCode}
 Estas se les conoce como vertex attributes y tienen un límite, como máximo 16 de vértices de 4 componentes.
\item Out:\+ sirven para mandar información al siguiente paso de renderizado. Como modificar la posición y luego pasarla. 
\begin{DoxyCode}{0}
\DoxyCodeLine{out\ vec4\ FragColor;}

\end{DoxyCode}

\item Uniforms:\+ son variables globales y que si valor no cambia hasta que se vuelve a enviar otro 
\begin{DoxyCode}{0}
\DoxyCodeLine{uniform\ vec4\ ourColor;}

\end{DoxyCode}

\end{DoxyItemize}\hypertarget{shaders_autotoc_md19}{}\doxysection{\texorpdfstring{3.\+2 Tipos}{3.\+2 Tipos}}\label{shaders_autotoc_md19}

\hypertarget{shaders_autotoc_md22}{}\doxysection{\texorpdfstring{3.\+4 Uniforms}{3.\+4 Uniforms}}\label{shaders_autotoc_md22}
Como ya hemos dicho antes, son variables globales y se pueden declarar en cualquier etapa de los shaders.

Para mandar información al shader, se hace de la siguiente manera 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{int}\ vertexColorLocation\ =\ glGetUniformLocation(shaderProgram,\ \textcolor{stringliteral}{"{}ourColor"{}});}
\DoxyCodeLine{glUniform4f(vertexColorLocation,\ v.x,\ v.y,\ v.z,\ v.w);\ \textcolor{comment}{//\ Espera\ un\ vec4\ de\ floats}}

\end{DoxyCode}


Primero se obtiene la localización de la variable en memoria, poniendo el nombre exacto de la variable. Siempre primero se tiene que hacer el use antes del uniform.
